## Overview

Rushed AI features a modern, browser-based code editor built on **CodeMirror 6**. The editor provides a professional development experience with syntax highlighting, a minimap for navigation, indentation markers, and intelligent code selection tools.

<Note>
The editor is optimized for web development with support for JavaScript, TypeScript, React (JSX/TSX), HTML, CSS, JSON, Markdown, and Python.
</Note>

## Editor Implementation

The editor is implemented as a React component with CodeMirror 6:

```tsx
import { EditorView, keymap } from "@codemirror/view";
import { oneDark } from "@codemirror/theme-one-dark";
import { indentWithTab } from "@codemirror/commands";
import { indentationMarkers } from "@replit/codemirror-indentation-markers";

const view = new EditorView({
  doc: initialValue,
  parent: editorRef.current,
  extensions: [
    ...themeExtensions,
    customSetup,
    languageExtension,
    quickEdit(fileName),
    selectionTooltip(fileName),
    keymap.of([indentWithTab]),
    minimap(),
    indentationMarkers(),
    EditorView.updateListener.of((update) => {
      if (update.docChanged) {
        onChange(update.state.doc.toString());
      }
    }),
  ],
});
```

## Language Support

The editor automatically detects file types and applies appropriate syntax highlighting:

```typescript
export const getLanguageExtension = (filename: string): Extension => {
  const ext = filename.split(".").pop()?.toLowerCase();

  switch(ext) {
    case "js":
      return javascript();
    case "jsx":
      return javascript({ jsx: true });
    case "ts":
      return javascript({ typescript: true });
    case "tsx":
      return javascript({ typescript: true, jsx: true });
    case "html":
      return html();
    case "css":
      return css();
    case "json":
      return json();
    case "md":
    case "mdx":
      return markdown();
    case "py":
      return python();
    default:
      return [];
  }
};
```

## Selection Tooltip

When you select code, a tooltip appears with two options:

```typescript
const createTooltipForSelection = (state: EditorState, fileName: string) => {
  const selection = state.selection.main;
  
  if (selection.empty) {
    return [];  // No selection
  }
  
  return [{
    pos: selection.to,
    above: false,
    create() {
      const dom = document.createElement("div");
      dom.className = "bg-popover text-popover-foreground z-50 rounded-sm border ...";
      
      // Add "Add to Chat" button
      const addToChatButton = document.createElement("button");
      addToChatButton.textContent = "Add to Chat";
      addToChatButton.onclick = () => {
        const selectedText = state.doc.sliceString(selection.from, selection.to);
        window.dispatchEvent(new CustomEvent("rushed:add-to-chat", {
          detail: { text: selectedText, fileName }
        }));
      };
      
      // Add "Quick Edit" button with ⌘E shortcut
      const quickEditButton = document.createElement("button");
      quickEditButton.textContent = "Quick Edit ⌘E";
      quickEditButton.onclick = () => {
        editorView.dispatch({
          effects: showQuickEditEffect.of(true)
        });
      };
      
      dom.appendChild(addToChatButton);
      dom.appendChild(quickEditButton);
      return { dom };
    }
  }];
};
```

## Auto-save Mechanism

Changes are automatically saved using a debounced approach:

```tsx
const DEBOUNCE_MS = 1500;
const timeoutRef = useRef<NodeJS.Timeout | null>(null);

<CodeEditor
  fileName={activeFile.name}
  initialValue={activeFile.content}
  onChange={(content: string) => {
    // Clear existing timer
    if (timeoutRef.current) {
      clearTimeout(timeoutRef.current);
    }
    
    // Set new timer
    timeoutRef.current = setTimeout(() => {
      updateFile({ id: activeFile._id, content });
    }, DEBOUNCE_MS);
  }}
/>
```

<Note>
The 1.5-second debounce prevents excessive database writes while still providing a responsive auto-save experience.
</Note>

## Theme System

The editor adapts to your system theme:

```typescript
const { resolvedTheme } = useTheme();

const themeExtensions = useMemo(() => {
  return resolvedTheme === "dark" 
    ? [oneDark, customTheme] 
    : [customTheme];
}, [resolvedTheme]);
```

- **Dark Mode**: Uses CodeMirror's `oneDark` theme plus custom styling
- **Light Mode**: Uses only custom theme with light color scheme
- **Auto-switching**: Updates when system theme changes

## File Breadcrumbs

The editor displays breadcrumb navigation for nested files:

```
src > components > ui > button.tsx
```

Breadcrumbs are built by traversing the parent chain:

```typescript
const path: { _id: string; name: string }[] = [];
let currentId: Id<"files"> | undefined = fileId;

while (currentId) {
  const file = await ctx.db.get("files", currentId);
  if (!file) break;
  
  path.unshift({ _id: file._id, name: file.name });
  currentId = file.parentId;
}

return path;  // ["src", "components", "ui", "button.tsx"]
```

## Performance Optimizations

<Steps>
  <Step title="Memoized Extensions">
    Language and theme extensions are memoized to prevent unnecessary re-renders.
  </Step>
  
  <Step title="Debounced Saves">
    Auto-save is debounced to reduce database writes during active editing.
  </Step>
  
  <Step title="Key-based Remounting">
    Editor remounts when switching files using the file ID as the React key.
  </Step>
  
  <Step title="Cleanup on Unmount">
    Editor view is properly destroyed when the component unmounts to prevent memory leaks.
  </Step>
</Steps>

```typescript
useEffect(() => {
  const view = new EditorView({ /* ... */ });
  viewRef.current = view;
  
  return () => {
    view.destroy();  // Cleanup
  };
}, [languageExtension, themeExtensions, fileName]);
```
