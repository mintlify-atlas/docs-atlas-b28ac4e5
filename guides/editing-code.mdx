## Overview

Rushed AI includes a powerful CodeMirror-based code editor with syntax highlighting, auto-completion, and modern editing features. While the AI can generate and modify code, the editor gives you full control for manual edits and fine-tuning.

## Opening Files

Files open in the editor when you click them in the file explorer.

<Steps>
  <Step title="Click a File">
    Single-click any file in the file explorer to open it in a temporary tab. The file content loads in the editor.
  </Step>

  <Step title="Pin the Tab (Optional)">
    Double-click a file to open it in a pinned tab. Pinned tabs stay open when you navigate to other files.
    
    <Tip>
      Single-clicked files open in temporary tabs that get replaced when you click another file. This keeps your workspace clean.
    </Tip>
  </Step>

  <Step title="View File Breadcrumbs">
    The breadcrumb navigation above the editor shows the full path to your current file:
    ```text
    src > components > Button.tsx
    ```
  </Step>
</Steps>

```typescript
// Opening files updates the editor state
const { openFile } = useEditor(projectId);

// Temporary tab (single click)
openFile(item._id, { pinned: false });

// Pinned tab (double click)
openFile(item._id, { pinned: true });
```

### Auto-Save

Changes save automatically after you stop typing:

<Note>
  **Debounce time: 1.5 seconds**
  
  The editor waits 1.5 seconds after your last keystroke before saving. This prevents excessive database writes while you type.
</Note>

```typescript
const DEBOUNCE_MS = 1500;

// Auto-save implementation
EditorView.updateListener.of((update) => {
  if (update.docChanged) {
    if (timeoutRef.current) {
      clearTimeout(timeoutRef.current);
    }
    
    timeoutRef.current = setTimeout(() => {
      updateFile({ id: activeFile._id, content });
    }, DEBOUNCE_MS);
  }
})
```

<Warning>
  Changes are saved locally. They don't affect any live deployment until you export or deploy your project.
</Warning>

## Binary Files

Some files can't be displayed in the text editor:

<Warning>
  Binary files (images, PDFs, etc.) and files with unsupported encodings display a warning message instead of content.
</Warning>

You'll see:
```text
The file is not displayed in the text editor because it is
either binary or uses an unsupported text encoding.
```

To work with these files:
- View images in the preview panel
- Download and edit locally
- Replace through file management

## Troubleshooting

### Syntax Highlighting Incorrect

If colors look wrong:
1. Check the file extension is correct
2. Rename the file if needed
3. Refresh the page to reload the editor

### Changes Not Saving

If edits seem to disappear:
1. Wait for the 1.5-second debounce period
2. Check your internet connection
3. Look for error messages in the browser console
4. Try refreshing and re-editing

### Editor Feels Slow

For very large files:
1. Consider breaking into smaller modules
2. Check browser performance
3. Close unnecessary tabs
4. Refresh the page

### Can't Select Text Properly

If selection behaves oddly:
1. Make sure you're clicking in the editor area
2. Try using keyboard selection (Shift + arrows)
3. Refresh the page if issues persist
