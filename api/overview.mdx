## Welcome to the Rushed AI API

Rushed AI provides two types of APIs for interacting with the platform:

1. **REST API Endpoints** - HTTP endpoints for project creation and GitHub export
2. **Convex Backend** - Real-time database queries and mutations for files, conversations, and project management

## Base URLs

<Note>
  Most of the Rushed AI platform runs on Convex, a real-time database with built-in API capabilities. Only a few operations use traditional REST endpoints.
</Note>

### REST API Endpoints
```
https://your-app.vercel.app/api
```

### Convex Backend
```
https://your-convex-deployment.convex.cloud
```

Your Convex deployment URL is available in your environment as `NEXT_PUBLIC_CONVEX_URL`.

## Making Requests

### REST API

For REST endpoints, send standard HTTP requests with JSON payloads:

```bash
curl -X POST https://your-app.vercel.app/api/projects/create-with-prompt \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_CLERK_TOKEN" \
  -d '{
    "prompt": "Create a todo app with React",
    "source": "dashboard"
  }'
```

### Convex Backend

For Convex operations, use the Convex client library:

```typescript
import { useQuery, useMutation } from "convex/react"
import { api } from "../convex/_generated/api"

// Query data
const projects = useQuery(api.projects.list)

// Mutate data
const createProject = useMutation(api.projects.create)
await createProject({ name: "My Project" })
```

## Authentication

All API requests require authentication via [Clerk](/api/authentication). See the [Authentication](/api/authentication) guide for details on obtaining and using tokens.

## Rate Limits

<Warning>
  Rate limits are enforced through the credit system. Each API operation consumes credits based on AI usage and resource consumption.
</Warning>

### Credit-Based Limits

- **Daily Credit Cap**: Varies by subscription plan (Starter: 10, Pro: 40, Scale: 150)
- **Daily Token Limit**: Varies by plan (Starter: 100k, Pro: 500k, Scale: 2M)
- **Model Costs**: Claude Sonnet 4.5 costs $3.00 per million input tokens, $15.00 per million output tokens

See [Credits](/resources/credits) for detailed pricing information.

## Error Handling

### REST API Errors

REST endpoints return standard HTTP status codes:

| Status Code | Description |
|-------------|-------------|
| `200` | Success |
| `400` | Bad Request - Invalid parameters |
| `401` | Unauthorized - Missing or invalid authentication |
| `403` | Forbidden - Insufficient credits or permissions |
| `404` | Not Found |
| `500` | Internal Server Error |

### Error Response Format

```json
{
  "error": "Error message",
  "code": "error_code",
  "details": {}
}
```

### Common Error Codes

### Convex Errors

Convex functions throw errors with descriptive messages:

```typescript
try {
  await createProject({ name: "" })
} catch (error) {
  // Error: "Name is required"
  console.error(error.message)
}
```

## Response Format

### REST API Responses

Successful REST responses return JSON:

```json
{
  "projectId": "k1234567890abcdef",
  "message": "Project created successfully"
}
```

### Convex Responses

Convex queries and mutations return strongly-typed data:

```typescript
// Query returns array of projects
const projects = await ctx.db.query("projects").collect()

// Mutation returns the created document ID
const projectId = await ctx.db.insert("projects", { name: "My Project" })
```

### React Integration

For React applications, use Convex React hooks:

```typescript
import { ConvexProvider, ConvexReactClient } from "convex/react"
import { api } from "../convex/_generated/api"

const convex = new ConvexReactClient(process.env.NEXT_PUBLIC_CONVEX_URL!)

function App() {
  return (
    <ConvexProvider client={convex}>
      <YourApp />
    </ConvexProvider>
  )
}
```
