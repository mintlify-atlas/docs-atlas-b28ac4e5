## Welcome to Rushed AI

Rushed AI is an AI-powered web application builder that runs entirely in your browser. Enter a natural language prompt, and watch as Claude Sonnet 4 generates a complete, working application with live preview and instant export to GitHub.

<Info>
  **What you'll build:** By the end of this guide, you'll have created your first AI-generated web application, previewed it in the browser, and learned how to iterate with AI assistance.
</Info>

## Prerequisites

All you need is:
- A modern web browser (Chrome, Firefox, Safari, or Edge)
- An internet connection
- A GitHub account (optional, for exporting your project)

<Note>
  No local development environment required. Rushed AI runs completely in your browser using WebContainer API.
</Note>

## Create Your Account

<Steps>
  <Step title="Sign Up with Clerk">
    Navigate to [rushed.ai](https://rushed.ai) and click **Start Building**. Rushed AI uses Clerk for secure authentication.
    
    You can sign up with:
    - Email and password
    - Google account
    - GitHub account (recommended for seamless export later)
  </Step>
  
  <Step title="Complete Registration">
    Follow the authentication flow to verify your account. You'll be automatically redirected to your dashboard after successful sign-up.
  </Step>
</Steps>

## Build Your First Project

<Steps>
  <Step title="Access the Dashboard">
    After signing in, you'll land on the dashboard at `/dashboard`. Here you'll see the project creation interface with three options:
    
    - **Prompt Input**: Describe what you want to build
    - **New**: Create a blank project
    - **Import**: Import from GitHub
  </Step>
  
  <Step title="Enter Your Prompt">
    In the prompt input field that says "Yo, tell Rushed what to build!", describe your application idea in natural language.
    
    <CodeGroup>
    ```text Beginner Examples
    Build a todo list app with dark mode
    
    Create a weather dashboard that shows current conditions
    
    Make a simple blog with markdown support
    ```
    
    ```text Advanced Examples
    Build a kanban board with drag-and-drop, local storage persistence, and multiple boards
    
    Create a pomodoro timer with customizable intervals, task tracking, and sound notifications
    
    Make a markdown editor with live preview, syntax highlighting, and export to PDF
    ```
    </CodeGroup>
    
    <Tip>
      Be specific about features, styling preferences, and functionality. The more detailed your prompt, the better the AI can understand your requirements.
    </Tip>
  </Step>
  
  <Step title="Click Build Now">
    Click the **Build now** button to start generating your project. Rushed AI will:
    
    1. Create a new project with an auto-generated name (e.g., "calm-dog-blue")
    2. Send your prompt to Claude Sonnet 4
    3. Generate the complete file structure and code
    4. Set up the preview environment
    
    This typically takes 15-45 seconds depending on project complexity.
  </Step>
</Steps>

## Explore the Editor Interface

Once generation completes, you'll be redirected to `/projects/{projectId}` where you'll see the full IDE experience:

### Code View

The editor is powered by CodeMirror and includes:

- **File Explorer** (left sidebar): Browse your project's file structure
- **Code Editor** (center): Edit files with syntax highlighting, autocomplete, and indentation markers
- **AI Chat** (right sidebar): Continue the conversation with Claude

<CodeGroup>
```typescript Keyboard Shortcuts
⌘K (or Ctrl+K)     Search projects
@filename          Tag files in chat
Select + Chat      Add code selection to chat context
⌘J (or Ctrl+J)     Create new blank project
⌘I (or Ctrl+I)     Import from GitHub
```

```typescript Editor Features
// Quick Edit: Select code and press the edit icon
// Selection Tooltip: Select code to see chat option
// Auto-save: Changes save automatically after 1.5 seconds
// Minimap: Overview of entire file (right edge)
// Indentation Markers: Visual guide for code blocks
```
</CodeGroup>

### Preview View

Click the **Preview** tab to see your application running live:

<Steps>
  <Step title="Wait for Container Boot">
    The WebContainer API boots a Node.js environment in your browser. You'll see:
    
    ```
    Starting container...
    Installing packages...
    ```
    
    This usually takes 10-30 seconds on first load.
  </Step>
  
  <Step title="View Your App">
    Once booted, your application loads at a local URL (e.g., `https://localhost:3000`). The preview shows:
    
    - Live application output
    - Real-time terminal logs (toggle with terminal icon)
    - Refresh button to restart the container
    - URL bar with copy functionality
  </Step>
  
  <Step title="Test Interactivity">
    Interact with your application directly in the preview pane. Any changes you make in the code editor will trigger a container restart to reflect updates.
  </Step>
</Steps>

<Warning>
  The WebContainer requires a modern browser with SharedArrayBuffer support. If you see errors, ensure your browser is up to date.
</Warning>

## Iterate with AI Chat

The conversation sidebar lets you refine your project through natural language:

<Steps>
  <Step title="Tag Files with @">
    Reference specific files in your prompts using the `@` symbol:
    
    ```
    @src/app/page.tsx add a header with navigation links
    ```
    
    As you type `@`, a file picker appears showing all project files. Select files to add them to context.
  </Step>
  
  <Step title="Select Code to Chat">
    In the code editor:
    
    1. Select any code block
    2. Click the chat icon in the selection tooltip
    3. The selected code automatically populates the chat with markdown formatting
    
    This gives the AI precise context about what you want to modify.
  </Step>
  
  <Step title="Send Your Request">
    Type your modification request and click **Send**. Examples:
    
    <CodeGroup>
    ```text Feature Additions
    Add authentication with login and signup pages
    
    Implement dark mode with a toggle in the header
    
    Add form validation with error messages
    ```
    
    ```text Bug Fixes
    The delete button isn't working, can you fix it?
    
    The layout breaks on mobile, make it responsive
    
    State isn't persisting on refresh, add localStorage
    ```
    
    ```text Styling Changes
    Make this look more modern with gradients
    
    Change the color scheme to match Tailwind's indigo
    
    Add animations when items are added to the list
    ```
    </CodeGroup>
  </Step>
  
  <Step title="Review AI Changes">
    The AI will:
    
    1. Analyze your request and tagged context
    2. Modify or create files as needed
    3. Provide a summary of changes made
    4. Update the preview automatically
    
    All changes appear in real-time in the file explorer and editor.
  </Step>
</Steps>

<Info>
  **Conversation History**: Click the clock icon in the chat header to view past conversations. Click the plus icon to start a new conversation thread.
</Info>

## Export to GitHub

When you're ready to deploy or collaborate:

<Steps>
  <Step title="Click Export Button">
    In the Code view, click the **Export** button in the top-right toolbar. This opens the GitHub export dialog.
  </Step>
  
  <Step title="Configure Repository">
    Fill in the repository details:
    
    ```tsx Repository Settings
    Repository Name: my-rushed-project
    Visibility: Private (or Public)
    Description: Built with Rushed AI (optional)
    ```
    
    <Note>
      Repository names must contain only alphanumeric characters, hyphens, underscores, and dots.
    </Note>
  </Step>
  
  <Step title="Create Repository">
    Click **Create Repository**. Rushed AI will:
    
    1. Create a new GitHub repository under your account
    2. Push all project files
    3. Set up a basic README
    4. Provide a direct link to the repository
    
    Export typically completes in 5-15 seconds.
  </Step>
  
  <Step title="View on GitHub">
    Once export completes, click **View on GitHub** to open your new repository. From there you can:
    
    - Clone it locally with `git clone`
    - Deploy to Vercel, Netlify, or other platforms
    - Share with collaborators
    - Continue development in your preferred IDE
  </Step>
</Steps>

<Warning>
  GitHub export requires:
  - Pro plan subscription
  - Connected GitHub account (via Settings)
  - Valid GitHub authentication token
</Warning>

## Understanding Credits

Rushed AI uses a credit system for AI operations:

- **Project creation from prompt**: Consumes credits based on complexity
- **AI chat messages**: Each message with Claude consumes credits
- **File modifications**: Larger changes consume more credits

You'll see credit warnings when running low:

```json Credit Error Response
{
  "error": "Unable to process request. Please check your subscription.",
  "code": "insufficient_credits"
}
```

Upgrade your plan or purchase credit packs at `/pricing` to continue building.

## Best Practices

<Tip>
  **Start Simple, Iterate Often**: Begin with a basic version of your app, then use the AI chat to incrementally add features. This gives you more control and better results than asking for everything at once.
</Tip>

<Tip>
  **Be Specific About Tech Stack**: If you want to use specific libraries or frameworks, mention them in your initial prompt:
  
  ```
  Build a todo app using React, Tailwind CSS, and Zustand for state management
  ```
</Tip>

<Tip>
  **Use @ Tags Liberally**: When requesting changes, tag relevant files with `@` to give the AI precise context. This dramatically improves accuracy.
</Tip>

<Tip>
  **Review Code Before Exporting**: Take a moment to review the generated code in the editor. You can manually edit any file before exporting to GitHub.
</Tip>

<Tip>
  **Save Your Conversations**: Each conversation is saved automatically. Use multiple conversation threads to explore different approaches to the same feature.
</Tip>

## Common Patterns

### Adding Authentication

```
@src/app/layout.tsx add Clerk authentication with sign in and sign up buttons in the header
```

### Making Responsive

```
Make the entire app responsive for mobile, tablet, and desktop using Tailwind breakpoints
```

### State Management

```
@src/app/page.tsx refactor to use Zustand for state management instead of useState
```

### Styling Updates

```
Update the design to use a modern glassmorphism style with backdrop blur and subtle gradients
```

### API Integration

```
Add integration with the OpenWeather API to fetch real-time weather data. Store the API key in .env.local
```
